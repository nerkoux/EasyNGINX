# EasyNginx docs site

Built with [Docusaurus](https://docusaurus.io/). Source for [the EasyNginx docs](https://easynginx.akshatmehta.com).

## Local development

```bash
cd web
npm install
npm start
```

Opens `http://localhost:3000/` and live-reloads on edits.

## Build

```bash
npm run build
npm run serve   # preview the production build at localhost:3000
```

The static output goes to `web/build/`.

## Deploy

Hosted on [Vercel](https://vercel.com/) at `easynginx.akshatmehta.com`.

The repo's `vercel.json` tells Vercel to:

- Set the project root to `web/`.
- Run `npm run build` on every push to `main` that touches `web/**`.
- Serve the static output from `web/build/`.

After connecting the repo in the Vercel dashboard once, every push deploys automatically. The custom domain `easynginx.akshatmehta.com` is configured under the project's Domains settings.

## Layout

```
web/
├── docs/                  # all documentation
│   ├── index.md           # landing page (slug: /)
│   ├── getting-started/   # install, quick start, existing-server, migrate
│   ├── commands/          # one page per command group
│   ├── guides/            # WordPress, Laravel, Node, Cloudflare, etc.
│   ├── operations/        # safety, backups, updates, cautions, troubleshooting
│   ├── architecture.md
│   └── changelog.md
├── src/
│   ├── components/HomepageFeatures/
│   ├── css/custom.css     # site theme
│   └── pages/index.tsx    # landing page hero
├── static/img/            # logos, social cards
├── docusaurus.config.ts
├── sidebars.ts
└── package.json
```

## Adding a page

1. Drop a markdown file in the right `docs/` subdirectory.
2. Add it to the corresponding entry in `sidebars.ts`.
3. `npm start` to preview.

The sidebar groups in `sidebars.ts` are explicit (not `autogenerated`) so the order is stable.
